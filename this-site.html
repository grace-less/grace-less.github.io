This Site

This being my first site, I tracked my progress for both a class and for myself. As I learn more about websites, I will update this to reflect my knowledge. -SGD

First HTML

Writing the first few lines in HTML was fairly easy. HTML is very straightforward, creating a habit of making tags for things before you start typing is the hard part.

First CSS

The CSS was a lot of fun, you get to play with the visual aspect of the site. I chose specific colors and tinkered with spacing a little bit. The hardest part is adjusting to a different form of measurements, like pixels.

First Bootstrap

This step irritated me the most. No matter how I played with the Basic Bootstrap, I could not get it to do what I wanted. It also broke all the CSS I had previously applied.

Template Source

Looking at templates, this seemed to be a good source for simplistic elements. I later downloaded a second template from this site to compare bootstrap files and tinker with elements. There are a lot of other sites with more elaborate templates but for what I wanted, this worked perfectly. I later learned you can also use these simplistic templates to pull individual elements from so you don't have to write them completely from scratch.

Template Base

I chose this Sidebar base because I like the functionality of a togglable menu. It also gave me the option of using a dropdown if I so desired.

First Edit Attempt

For a 20 page website, not including the main index page, I found that putting them all on one menu list was too much. Also, putting them into categorized lists was not visually pleasing.

Second Edit Attempt

I attempted to remedy the visual mess by turning the categories into individual dropdown menus. It got closer to the desired effect but still looked sloppy.

Third Edit Attempt

I was able to turn them into stacked buttons that slide down to expose more buttons, in web development the element is called accordion which I had to look into off of the w3 schools website. Unfortunately, while it gave me the effect I wanted, the design was still flawed.

Finally Got It

After adjusting button sizes and spacing, I got the desired look as well as effect. It took a couple hours to find what was throwing the buttons off and under which element but alas the margin was found.

Finding Color

I discovered the template had all the colors buried in the Bootstrap and a unfortunate side-effect of VScode's delightfully dark background is the inability to destinguish between white and off-white. So I changed the palette to grayscale and noted the hex codes so I could find them much easier when needed.

Adding Content

Adding the content was the easy part. I made a powerpoint consisting of 1 slide per web-page and titled them like my pages. Then wrote a paragraph for each page to be copied and pasted into the corresponding webpages.

Adding a Place For Pictures

The second template I mentioned downloading earlier had a link to a picture placeholder on it, so I copied it into my website and postitioned it where I wanted to keep things simple.

Deciding Colors

This was one of the hardest parts, simply because I'm uncoordinated when it comes to color. Luckily I have artsy friends that I could turn to and discuss the technical issues certain colors, shades, and hues may create on a webpage versus a canvas.

What You See

I've spent weeks pouring over code and reference materials but I made it. It's a finished product, at least, it can be considered one. I pulled what pictures I had that I could use and found others that fit what I was going for. Then I plugged them into the space I had set aside for them, created a FavIcon to display as well, and combed through for any snags or hicccups I might have. I know that to some degree, it still needs a lot of TLC but I feel like I don't have the depth of knowledge to fix some of it just yet. However, I am proud of it as a whole and look forward to adding to and expanding this over time.
